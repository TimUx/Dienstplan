1. Einführung

Das Dienstplan-System unterstützt Sie bei der Verwaltung von Schichtplänen für Mitarbeiter in mehreren Teams.

Abbildung 1: Hauptansicht des Dienstplans (Wochenansicht)

Hauptfunktionen:

  • Automatische Schichtplanung: Das System erstellt automatisch Schichtpläne unter Beachtung aller gesetzlichen und betrieblichen Regeln
  • Mitarbeiterverwaltung: Verwalten Sie Mitarbeiter, Teams und Abwesenheiten mit erweiterten Daten (E-Mail, Geburtsdatum, Funktion)
  • Urlaubsverwaltung: Mitarbeiter können Urlaubsanträge einreichen, die vom Disponent genehmigt werden
  • Diensttausch-System: Mitarbeiter können Dienste untereinander tauschen (mit Genehmigung)
  • Springer-System: Automatische Zuweisung von Vertretungen bei Ausfällen
  • Statistiken: Übersicht über Arbeitsstunden, Schichtverteilung, Wochenend-Dienste und Auslastung
  • E-Mail-Benachrichtigungen: Konfigurierbare E-Mail-Benachrichtigungen für alle Beteiligten

2. Anmeldung

Für die Nutzung des Systems benötigen Sie einen Benutzeraccount mit entsprechenden Berechtigungen:

  • Administrator: Volle Berechtigung für alle Funktionen
  • Disponent: Schichtplanung und Bearbeitung
  • Mitarbeiter: Nur Lesezugriff auf eigenen Dienstplan

Abbildung 2: Anmeldedialog mit Rollenbasierter Zugriffskontrolle

Hinweis: Wenden Sie sich an Ihren Administrator, um einen Account zu erhalten.

3. Dienstplan verwalten

Dienstplan anzeigen

Wählen Sie im Hauptmenü "Dienstplan" aus. Sie können zwischen folgenden Ansichten wählen:

  • Woche: Detaillierte Ansicht für 7 Tage
  • Monat: Übersicht für 30 Tage
  • Jahr: Jahresübersicht

Abbildung 3a: Dienstplan in der Wochenansicht (KW 50 2025)

Abbildung 3b: Dienstplan in der Monatsansicht (Dezember 2025)

Abbildung 3c: Dienstplan in der Jahresansicht (2025)

Schichten automatisch planen

  1. Klicken Sie auf "Schichten planen"
  2. Wählen Sie den Zeitraum aus
  3. Bestätigen Sie die Planung

Das System erstellt automatisch einen optimalen Schichtplan unter Beachtung aller Regeln.

Manuelle Anpassungen

Sie können jederzeit manuelle Änderungen am Dienstplan vornehmen:

  • Einzelne Schichten können individuell zugewiesen werden
  • Manuelle Zuweisungen werden vom System respektiert und nicht überschrieben

4. Mitarbeiter verwalten

Abbildung: Mitarbeiterverwaltung

Neuen Mitarbeiter anlegen

  1. Wechseln Sie zur Ansicht "Mitarbeiter"
  2. Klicken Sie auf "+ Mitarbeiter hinzufügen"
  3. Füllen Sie die Pflichtfelder aus:
    • Vorname
    • Name
    • Personalnummer (eindeutig)
  4. Optionale Felder:
    • E-Mail: Für Benachrichtigungen
    • Geburtsdatum: Für Personalverwaltung
    • Funktion: Z.B. Brandmeldetechniker, Brandschutzbeauftragter
    • Springer: Markieren für Vertretungspool
  5. Klicken Sie auf "Speichern"

Abwesenheiten erfassen

Erfassen Sie Abwesenheiten direkt über die API oder das Mitarbeiter-Profil:

  • Krank: Krankheitstage
  • Urlaub: Geplanter Urlaub (alternativ über Urlaubsantragssystem)
  • Lehrgang: Schulungen und Weiterbildungen

5. Urlaubsverwaltung

Das System bietet ein vollständiges Urlaubsantragssystem mit Workflow-Management.

Urlaubsantrag stellen (Mitarbeiter)

  1. Verwenden Sie die API: POST /api/vacationrequests
  2. Geben Sie Startdatum und Enddatum an
  3. Optional: Fügen Sie eine Notiz hinzu
  4. Der Status wird zunächst auf "In Bearbeitung" gesetzt

Urlaub genehmigen/ablehnen (Disponent/Admin)

  1. Rufen Sie offene Anträge ab: GET /api/vacationrequests/pending
  2. Prüfen Sie den Antrag
  3. Setzen Sie den Status auf "Genehmigt" oder "Nicht genehmigt": PUT /api/vacationrequests/{id}/status
  4. Optional: Fügen Sie eine Antwort/Begründung hinzu

Hinweis: Genehmigte Urlaubsanträge werden automatisch als Abwesenheiten vom Typ "Urlaub" in den Kalender eingetragen.

6. Diensttausch

Mitarbeiter können Dienste untereinander tauschen, mit Genehmigungspflicht durch den Disponent.

Dienst zum Tausch anbieten (Mitarbeiter)

  1. Verwenden Sie die API: POST /api/shiftexchanges
  2. Geben Sie die Schicht-ID an, die getauscht werden soll
  3. Optional: Geben Sie einen Grund an
  4. Der Dienst wird als "Angeboten" markiert

Angebotenen Dienst anfragen (Mitarbeiter)

  1. Verfügbare Tauschangebote anzeigen: GET /api/shiftexchanges/available
  2. Gewünschten Dienst anfragen: POST /api/shiftexchanges/{id}/request
  3. Status wechselt zu "Angefragt"

Tausch genehmigen (Disponent/Admin)

  1. Offene Tauschanfragen anzeigen: GET /api/shiftexchanges/pending
  2. Tausch prüfen und genehmigen/ablehnen: PUT /api/shiftexchanges/{id}/process
  3. Bei Genehmigung wird die Schicht automatisch umgeschichtet

Wichtig: Auch bei genehmigten Tauschen werden die Ruhezeiten und Planungsregeln geprüft!

7. Statistiken

Abbildung: Statistiken und Auswertungen

Die Statistik-Ansicht bietet Ihnen einen Überblick über:

  • Arbeitsstunden: Gesamtstunden und Schichtanzahl pro Mitarbeiter
  • Schichtverteilung: Verteilung der Schichtarten pro Team
  • Fehltage: Übersicht der Abwesenheiten nach Typ
  • Team-Auslastung: Durchschnittliche Schichten pro Mitarbeiter
  • Wochenend-Statistiken (Admin/Disponent): Samstags- und Sonntagsdienste je Mitarbeiter

Wählen Sie den gewünschten Zeitraum aus und klicken Sie auf "Aktualisieren".

Wochenend-Statistiken abrufen

Als Administrator oder Disponent können Sie detaillierte Wochenend-Statistiken abrufen:

GET /api/statistics/weekend-shifts?startDate=2025-01-01&endDate=2025-12-31

8. E-Mail-Einstellungen

Als Administrator können Sie die E-Mail-Server-Einstellungen konfigurieren, um automatische Benachrichtigungen zu ermöglichen.

E-Mail-Konfiguration anlegen

  1. Verwenden Sie die API: POST /api/emailsettings
  2. Erforderliche Einstellungen:
    • SMTP Server: Hostname oder IP-Adresse (z.B. smtp.gmail.com)
    • Port: Standard 587 für STARTTLS, 465 für SSL, 25 für unverschlüsselt
    • Protokoll: SMTP oder SMTPS
    • Sicherheit: None, SSL, TLS oder STARTTLS
    • Authentifizierung: Ja/Nein
    • Benutzername: Falls Authentifizierung erforderlich
    • Passwort: Falls Authentifizierung erforderlich
    • Absenderadresse: z.B. dienstplan@fritzwinter.de
    • Absendername: z.B. "Dienstplan-System"
    • Antwortadresse: Optional, falls abweichend

Mitarbeiter-E-Mails pflegen

Erfassen Sie E-Mail-Adressen für jeden Mitarbeiter, der Benachrichtigungen erhalten soll:

  1. Öffnen Sie den Mitarbeiter zur Bearbeitung
  2. Tragen Sie die E-Mail-Adresse im Feld "E-Mail" ein
  3. Speichern Sie die Änderungen

Aktive Konfiguration abrufen

GET /api/emailsettings/active

Sicherheitshinweis: Das System zeigt Passwörter niemals in API-Antworten an. Passwörter sollten vor der Speicherung verschlüsselt werden.

9. Schichtplanungsregeln

Das System beachtet automatisch folgende Regeln:

Schichtarten

Schicht Code Uhrzeit
Frühdienst F 05:45 – 13:45
Spätdienst S 13:45 – 21:45
Nachtdienst N 21:45 – 05:45
Zwischendienst ZD 08:00 – 16:00

Besetzungsregeln

Montag bis Freitag:

  • Frühdienst: 4-5 Personen
  • Spätdienst: 3-4 Personen
  • Nachtdienst: 3 Personen

Samstag und Sonntag:

  • Alle Schichten: maximal 3 Personen

Verbotene Schichtwechsel

  • ❌ Spätdienst → Frühdienst (zu kurze Ruhezeit)
  • ❌ Nachtdienst → Spätdienst (zu kurze Ruhezeit)
  • ❌ Gleiche Schicht zweimal hintereinander

Idealer Rhythmus

Frühdienst → Nachtdienst → Spätdienst

Dieser Rhythmus ermöglicht optimale Ruhezeiten und Belastungsverteilung.

Ruhezeiten

Mindestens 11 Stunden Ruhezeit zwischen zwei Schichten (gesetzlich vorgeschrieben).

10. Springer-System

Das Springer-System sorgt für automatische Vertretung bei kurzfristigen Ausfällen.

Springer definieren

Markieren Sie beim Anlegen oder Bearbeiten eines Mitarbeiters das Feld "Springer".

Automatische Zuweisung

Bei Erfassung einer Abwesenheit prüft das System automatisch:

  1. Welche Schichten betroffen sind
  2. Welche Springer verfügbar sind
  3. Weist den ersten verfügbaren Springer zu

Hinweis: Springer werden nur bei tatsächlichen Ausfällen eingesetzt, nicht bei der regulären Planung.

11. Problembehebung

Problem: Schichtplanung schlägt fehl

Lösung:

  • Prüfen Sie, ob genügend Mitarbeiter verfügbar sind
  • Überprüfen Sie bestehende Abwesenheiten
  • Stellen Sie sicher, dass mindestens 12-15 Mitarbeiter angelegt sind

Problem: Kein Springer verfügbar

Lösung:

  • Definieren Sie weitere Mitarbeiter als Springer
  • Prüfen Sie Abwesenheiten der Springer
  • Bei Bedarf: Manuelle Zuweisung einer Vertretung

Problem: Statistiken werden nicht angezeigt

Lösung:

  • Prüfen Sie den ausgewählten Zeitraum
  • Stellen Sie sicher, dass Schichten für diesen Zeitraum vorhanden sind
  • Aktualisieren Sie die Seite (F5)
